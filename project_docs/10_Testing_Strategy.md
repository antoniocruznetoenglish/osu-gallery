## 3. Regression Policy

- A bug fix ships with a test that reproduces the bug first (red), then passes once fixed (green). No fix without a regression test, unless explicitly justified in the PR/commit message.
- Tests are not deleted to make a build pass. A failing test is either fixed, or the feature it tests is deliberately removed (with a Decision Log or Feature Log note explaining why).
